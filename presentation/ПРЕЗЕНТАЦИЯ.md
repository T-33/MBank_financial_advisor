# MBank AI Финансовый Советник — Структура Питча для Жюри

## Контекст
Прототип, созданный за 24 часа на хакатоне. Питч представляет три ключевых функции (Выбор персоны, Перехват трат, AI Автопилот-накопления), подкреплён реальными отраслевыми метриками, предлагает MBank конкретный план разработки (время/стоимость/оборудование) и добавляет один новый демонстрируемый сценарий — AI-кросс-продажи внутренних сервисов MBank (MMarket, MProfi) при упоминании пользователем внешней покупки.

---

## Слайды (10–12 слайдов, ~3 мин демо + 2 мин Q&A)

### Слайд 1 — Хук (15 сек)
- Одна фраза: **«Мы построили не очередной PFM. Мы построили агента, который действует раньше тебя.»**
- Скриншот: баннер «Токсичного бро» — Перехват трат поверх главного экрана.

### Слайд 2 — Проблема
- Банковские приложения Кыргызстана (включая MBank сегодня) = транзакционные. Пользователь открывает → платит → закрывает.
- DAU/MAU низкий, потому что нет причины открывать приложение без конкретной задачи.
- Стандартные PFM-дашборды (круговые диаграммы, разбивка по категориям) имеют менее 5% вовлечённости по всей отрасли.
- **«Не показывай мне радар — скажи, нужен ли зонт.»**

### Слайд 3 — Наши три ключевых функции (основа демо)
1. **Выбор персоны** — Заботливый vs Токсичный бро. Вирусный слой.
2. **Перехват трат** — агент говорит первым, блокирует плохие транзакции до авторизации.
3. **AI Автопилот-накопления** — копилка, которой управляет AI сам; и это ЕСТЬ MSmart Депозит (8% годовых).
- Плюс (новое): **Сценарий внутреннего кросс-продажа** (см. Слайд 7).

### Слайд 4 — Сценарий живого демо (90 сек)
1. Открываем приложение → виджет Автопилота уже на главном экране (84 500 С / 500 000 С, цель «На машину», которую пользователь не ставил).
2. Нажимаем кнопку разработчика → Перехват трат выезжает сверху: *«3-й Тулпар за неделю…»* → нажимаем «Отмена — в копилку» → виджет тикает вверх с анимацией пульса.
3. Открываем чат → переключаем персону на Токсичного бро → спрашиваем «куда ушли деньги?» → карточка, сгенерированная через tool-call, рендерится прямо в пузыре сообщения.
4. Запускаем новый сценарий кросс-продажи (Слайд 7).
5. Запоминающаяся фраза-роаст остаётся на экране в финале.

### Слайд 5 — Почему это побеждает: Рыночные доказательства (цифры и метрики)
Реальные кейсы внедрения conversational AI / агентных советников в банкинге:
- **Bank of America — Erica**: более 2 миллиардов клиентских взаимодействий, 42 млн+ пользователей, рост использования 56% год к году (раскрытие BofA Q4'23).
- **Commonwealth Bank of Australia — Ceba / AI-ассистент**: снижение нагрузки на колл-центр на ~40%, экономия ~AUD 800M операционных затрат (годовой отчёт CBA FY23).
- **Capital One — Eno**: ~20% мобильно-активных клиентов используют ежемесячно; существенный рост DAU в мобайле.
- **Cleo (UK-необанк, доказательство концепции токсичного бро)**: перешёл 5 млн пользователей почти исключительно за счёт вирусности в TikTok/Instagram от скриншотов «роаст меня» — персона И ЕСТЬ канал привлечения.
- **Personetics (white-label PFM-AI для RBC, U.S. Bank, Intesa)**: зафиксированы **+35% вовлечённости в приложение**, **+27% удержания**, **4x использование PFM-функций** vs статические дашборды.
- **McKinsey 2024 Gen-AI в банкинге**: conversational-агенты прогнозируют добавить **$200–340 млрд/год** к глобальной производительности банкинга; ранние внедренцы видят **9–15% рост кросс-продаж цифровых продуктов**.

**Перевод на язык MBank**: консервативный рост DAU на 10% при ~1 млн активных пользователей = ~100К новых ежедневных сессий = кратно больше поверхности для кросс-продаж MMarket, MProfi, MSmart Депозита.

### Слайд 6 — История о доходах
- Банка Автопилота = MSmart Депозит, открытый тихо. Пользователю 8% годовых; MBank зарабатывает спред на размещённом капитале.
- Перехваченные транзакции = приток депозитов. Даже при 17 С средних × 3 перехвата/неделю × 500К пользователей ≈ **1,3 млрд С/год нового дешёвого фондирования**.
- Кросс-продажи (Слайд 7) направляют GMV в MMarket без затрат на рекламу.
- Рост DAU (~10%) улучшает все нижестоящие KPI, которые MBank уже измеряет.

### Слайд 7 — НОВОЕ: Сценарий внутреннего кросс-продажа (MMarket / MProfi)
**Питч в одной фразе**: *«Когда пользователь рассказывает боту, что купил, бот говорит, где надо было купить — внутри MBank.»*

**Сценарий демо:**
- Пользователь (в чате): *«Купил лампу на Орто-Сай за 500 С.»*
- Агент (токсичный режим): *«500 С за лампу? Та же самая в MMarket стоит 350 С и доставка бесплатная. В следующий раз спроси меня ДО того, как достать кошелек.»*
- Рендерится карточка через tool-call в пузыре: фото товара, **350 С** vs 500 С зачёркнутые, CTA «Открыть в MMarket ›».
- Вариант в режиме «Заботливый»: вежливый рефрейминг, та же карточка.

**Как это работает (для жюри, которые спросят):**
- Инструмент: `find_in_mbank_catalog(query, paid_price)` — каталог MMarket (замокан для демо, реальный API в проде).
- Та же схема покрывает **MProfi**: «нанял мастера за X» → предложи мастера MProfi по меньшей цене.
- Работает для MTravel, страховки и т.д. Один инструмент, N вертикалей.

**Почему это убийственное дополнение:**
- Превращает чат в **воронку привлечения с нулевым CAC** для маркетплейс-продуктов MBank.
- Каждое сообщение в чате — потенциальное GMV-событие.
- Вторая независимая линия дохода наряду с депозитным Автопилотом.

### Слайд 8 — План разработки для MBank (время, стоимость, оборудование)
**Объём**: перевести прототип в production в реальном мобильном приложении MBank.

**Команда и сроки:**
| Фаза | Длительность | Состав |
|---|---|---|
| Discovery + проверка рисков/compliance | 3 недели | 1 PM, 1 compliance, 1 solution architect |
| Чат + интеграция tool calling | 4 недели | 2 backend, 1 ML-инженер |
| Персона + Перехват трат + Автопилот | 4 недели | 2 mobile (iOS+Android), 1 дизайнер |
| Внутренний кросс-продажа (MMarket/MProfi) | 2 недели | 1 backend, 1 mobile |
| QA, security pen-test, поэтапный rollout | 3 недели | 1 QA, 1 security |
| **Итого MVP до прода** | **~16 недель (4 месяца)** | ~8 FTE в пике |

**Оценка стоимости (рыночные ставки Кыргызстана, USD):**
- Разработка (8 FTE × 4 месяца × ~$3 500 blended) ≈ **$112К**
- LLM-инференс (Claude Haiku / GPT-4o-mini mix, ~500К DAU, ~3 сообщения/пользователь/день, кэшированные промпты) ≈ **$8–15К/месяц** в steady state
- Единоразовая настройка (vector DB, prompt ops, evals, observability) ≈ **$20К**
- Security-аудит + compliance НБКР ≈ **$25К**
- **За первый год итого: ~$250–300К**
- Окупаемость: рост притока депозитов на 0,5% → окупится менее чем за 6 месяцев.

**Оборудование / инфраструктура:**
- **Никаких on-prem GPU для MVP.** Используем hosted LLM API (Anthropic / OpenAI / Yandex GPT как RU-резерв).
- Существующий Kubernetes-кластер MBank хостит слой оркестрации (tool-calling шлюз, хранилище сессий, PII redaction прокси).
- Добавляем: Redis (состояние чат-сессий), Postgres-таблица (конфиг персоны/автопилота), pgvector (RAG-каталог MMarket).
- **On-prem вариант (требования по хранению данных / давление НБКР позже)**: 2× NVIDIA L40S или 1× H100 поднимают квантизированный Llama-3.1-70B или Qwen2.5-72B при ~50 одновременных пользователях. CapEx ~$40–60К. Не нужен на старте.

### Слайд 9 — Риски и митигации
- **PR-риск токсичного режима** → только opt-in, жёсткий блок-лист категорий (медицина/аптеки/похороны), серверный safety-фильтр.
- **Галлюцинации в финансовых советах** → все цифры берутся из структурированных данных через tool call, никогда не из свободного текста LLM.
- **Падение LLM** → копия Перехвата трат + Автопилота захардкожена как fallback; агент деградирует до rule-based.
- **Compliance** → никакие PII не покидают периметр MBank; вызовы LLM проходят через redaction-прокси.
